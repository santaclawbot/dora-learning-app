## Executive Summary

"Ask Dora" is a conversational AI feature that lets kids ask questions and get friendly, educational responses in Dora's voice. It integrates OpenClaw's Dora agent for personality-consistent responses with ElevenLabs TTS for voice output.

## 2. API Design

### Endpoint: Send Message to Dora

```
POST /api/ask-dora/message
Authorization: Bearer <jwt_token>
Content-Type: application/json
```

**Request:**
```json
{
  "message": "Why is the sky blue?",
  "profileId": "aiden",
  "conversationId": "conv_abc123",       // optional, for context
  "generateAudio": true                   // optional, default true
}
```

**Response (Success):**
```json
{
  "success": true,
  "response": {
    "id": "msg_xyz789",
    "text": "Great question, friend! The sky looks blue because of something super cool called light scattering. When sunlight comes down from space, it bumps into tiny bits of air and bounces around like a bouncy ball! Blue light bounces the most, so that's what we see everywhere we look up! ğŸ’™",
    "audioUrl": "/audio/msg_xyz789.mp3",
    "timestamp": "2026-02-06T17:30:00Z"
  },
  "conversationId": "conv_abc123",
  "source": "openclaw"                    // or "claude-fallback"
}
```

**Response (Error):**
```json
{
  "success": false,
  "error": "AI service temporarily unavailable",
  "fallbackMessage": "Oops! Dora is taking a little nap. Try again in a moment! ğŸ˜´"
}
```

### Endpoint: Get Conversation History

```
GET /api/ask-dora/history?profileId=aiden&limit=20
Authorization: Bearer <jwt_token>
```

**Response:**
```json
{
  "conversations": [
    {
      "id": "conv_abc123",

      ],

}
```

### Endpoint: Start New Conversation

## 6. Database Schema Addition

```sql
-- New table for Ask Dora conversations
CREATE TABLE IF NOT EXISTS dora_conversations (
  id TEXT PRIMARY KEY,
  profile_id TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- Individual messages in conversations
CREATE TABLE IF NOT EXISTS dora_messages (
  id TEXT PRIMARY KEY,
  conversation_id TEXT NOT NULL,
  role TEXT NOT NULL,           -- 'user' or 'dora'
  content TEXT NOT NULL,
  audio_url TEXT,               -- For Dora's responses
  source TEXT,                  -- 'openclaw' or 'claude-fallback'
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY(conversation_id) REFERENCES dora_conversations(id)
);

-- Index for fast lookups
CREATE INDEX IF NOT EXISTS idx_messages_conversation 
  ON dora_messages(conversation_id, created_at);
```
